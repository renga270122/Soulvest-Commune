# Soulvest Commune Pilot Walkthrough

## Demo Story

1. Resident creates a visitor pass or guard logs a walk-in.
2. Resident receives an in-app approval alert.
3. Guard checks the visitor in and later checks them out.
4. Admin posts a pinned water shutdown notice.
5. Resident acknowledges the notice with the Got It button.
6. Admin seeds demo data to show one dues cycle and one active complaint.
7. Resident checks dues in the payments screen.
8. Resident asks the AI concierge about due status or complaint status.
9. Admin exports dues CSV for committee review.

## Dry Run Checklist

1. Sign in once as admin and seed demo day data.
2. Sign in as resident and verify notice acknowledgment, complaint status, and dues insight.
3. Sign in as guard and run check-in and check-out on a visitor.
4. Confirm that notifications appear in the resident dashboard.
5. Confirm that announcements, complaints, and dues reflect the same demo state across roles.

## Resident Onboarding Notes

- Use the resident dashboard for visitor approvals and notifications.
- Use the announcements page to acknowledge important notices.
- Use the payments page to view current dues and payment history.
- Use the complaints page to raise and track maintenance issues.
